How this actually works

Live notes on the real system, not a mockup anymore.

Pricing

Prices live in a real SQLite database on the API server. Changes here call PUT /api/admin/countries/:id and take effect immediately across the public site — there's no separate "publish" step.

How AI document review actually runs

  • Customer uploads a file on the public site → sent to the API as base64 with an application ID and document type.
  • The API calls Claude with a prompt describing the rule for that document type, and asks for a strict pass / needs_review / failed verdict.
  • Anything not a clean "passed" lands in this queue for you to resolve — the AI never auto-files or auto-rejects on its own.
  • The Anthropic API key lives only in the server's environment variables — never sent to any browser.

Still to do before this handles real customers

  • Document files themselves aren't stored anywhere yet — only the AI's verdict is saved. Add real file storage (e.g. S3-compatible bucket) before going live.
  • No rate limiting on login/forgot-password yet — add this before public launch.
  • Password reset codes are logged server-side, not emailed — wire up a real email provider.
  • Render's free tier resets its disk on redeploy unless you attach a persistent disk — do this before trusting it with real data.
